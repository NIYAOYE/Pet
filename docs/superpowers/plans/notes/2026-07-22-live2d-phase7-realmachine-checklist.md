# Live2D Phase 7 真机验收清单

代码/自动化部分已完成(`pnpm typecheck && pnpm test && pnpm build` 全绿),以下必须真机人工验证,agent 会话无法自动化(无真实 Live2D 模型/无法模拟多显示器与 DPI 缩放/无 Intel-AMD-NVIDIA 多卡对比条件)。

范围依据(2026-07-22 与用户确认):设计文档 §15.4 原矩阵包含 Windows 10/11、100%/150%/200% DPI、单屏/双屏不同 DPI、Intel/AMD 集成显卡与 NVIDIA 独显对比、跨屏拖动、显示器休眠/锁屏恢复。用户目前只有一台机器,能覆盖:单屏(当前这台机器的实际 DPI/显卡)、同一台机器切换 Windows 缩放(100%/150%/200%)、双屏(不同 DPI)。**不覆盖**多台不同 GPU 厂商机器的对比——这条从矩阵里划掉,不是"待办",是范围外。

## 准备

1. 确认本地有至少一个真实授权的 Live2D 模型(`*.model3.json`),按应用内"导入 Live2D 模型"流程导入成 `pets/<id>/` 包。
2. `pnpm build && pnpm preview`(或安装版,见下方"安装版 vs 开发版"一节)启动应用,切到该 live2d 宠物。

## A. GPU Context Lost 恢复(承接 `docs/superpowers/plans/2026-07-22-live2d-phase7-gpu-context-recovery.md` Task 4 Step 2/3,本次 agent 会话因沙箱无真实 Live2D 模型未能执行)

- [ ] DevTools Console 执行 `window.__kiboLive2D.app.renderer.gl.getExtension('WEBGL_lose_context').loseContext()`,确认:画面消失,出现"画面渲染出现问题,正在尝试恢复…"红底文字占位;期间点击宠物区域仍能弹出/收起对话框。
- [ ] 执行 `.restoreContext()`,确认:占位消失,模型重新出现并正常播放待机动作。
- [ ] 连续两次 `loseContext()`(不等待中间 restore 完成),确认:占位文案变为"渲染反复失败,已停止自动重试。请从托盘或设置中切换宠物/模型。",此后 `restoreContext()` 不再恢复模型(given-up 终态)。
- [ ] 上一步之后,从托盘/设置切换到另一个宠物,确认切换正常、新宠物不受旧宠物 given-up 状态影响。
- [ ] 真实触发一次(非 DevTools 模拟)GPU 上下文丢失——最方便的是让屏幕/显示器进入休眠再唤醒,或触发一次显卡驱动重置(如果有条件)——确认同样走上面 A 组的恢复路径,不是只有人为模拟才有效。

## B. DPI / 多屏矩阵

- [ ] 单屏,当前这台机器实际 DPI 缩放下:窗口尺寸、鼠标追踪、命中检测(点击/双击/拖拽/透明区域穿透)均正确。
- [ ] Windows 设置里把系统缩放依次切到 100%、150%、200%(每次切换后可能需要重新登录或至少重启应用),确认三档下宠物窗口尺寸、气泡窗口位置、鼠标追踪目标计算(`computeMouseFocusTarget`)均无错位或比例失真。
- [ ] 双屏、两块屏幕设置不同 DPI 缩放:把宠物从低 DPI 屏拖到高 DPI 屏(以及反方向),确认跨屏拖动时窗口尺寸/清晰度/鼠标追踪没有用错来源屏幕的 DPI 值。
- [ ] 双屏环境下,宠物停留在非主屏时对话框/气泡窗口定位正常(不跑到另一块屏幕上)。

## C. 常规交互回归(确认 Phase 7 改动没有引入回归)

- [ ] 点击穿透:模型像素上不穿透,透明区域穿透(含 GPU 恢复流程之外的正常状态)。
- [ ] 拖拽移动窗口跟手,行走状态左右镜像正确。
- [ ] 单击开/关对话框、双击 poke,与 Phase 6 之前行为一致。
- [ ] 显示器休眠再唤醒 / 锁屏再解锁:宠物窗口能正常恢复显示(不只是 GPU context——也确认 Electron 窗口本身、Ticker 可见性节流没有卡死)。

## D. 安装版 vs 开发版

- [ ] `pnpm dist` 打包一次,安装后跑一遍 A/B/C 组的关键项(至少:GPU 恢复 DevTools 模拟一次、单屏正常渲染、点击穿透),确认打包产物行为与 `pnpm preview` 一致。

## 反馈

把每一项的结果(通过/不通过 + 具体现象)发回来,不通过的项目附上 console 报错和肉眼观察到的现象(黑屏/花屏/卡顿/贴图错位/占位文案不对等)。
